# AWS SOC Architecture

## Objective
Document the cloud security monitoring architecture used in this SOC laboratory.

---

# Architecture Overview

This project simulates a cloud security monitoring environment using AWS native security services.

The architecture was designed to detect suspicious activities, generate alerts, and support incident response workflows.

---

# Services Used

## AWS CloudTrail
Responsible for recording AWS account activities and API events.

Examples:
- Console logins
- IAM changes
- Root account usage
- Security Group modifications

---

## Amazon CloudWatch
Responsible for monitoring CloudTrail metrics and triggering alarms.

Examples:
- Failed login detection
- Root account activity alerts
- IAM user creation alerts

---

## Amazon SNS
Responsible for sending email notifications generated by CloudWatch alarms.

Examples:
- Security alerts
- Monitoring notifications
- Incident detection emails

---

# Detection Flow

```text
User Activity
      ↓
      AWS CloudTrail
            ↓
            CloudWatch Metrics
                  ↓
                  CloudWatch Alarm
                        ↓
                        Amazon SNS
                              ↓
                              Email Notification
                              ```
